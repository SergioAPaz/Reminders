Falso arranque rapido en Windows 10

Windows 10 vs Linux

Ya de hace tiempo hemos leído lo mucho que presume Windows 10 acerca del famoso arranque rápido con el que cuenta Windows en especial en esta ultima y mas reciente actualización a windows 10 anniversary update pero que hay de cierto en el?

Pues al parecer no mucho...


Por que si bien aparentemente Windows 10 si arranca mas rapido (Según Microsoft) que por cierto yo tengo una PC de ultima generación con 8 GB de RAM y un procesador Intel I5 Sexta generación no me parece que sea un arranque sobresaliente a mi consideración en especial comparándolo con Linux Mint 18 "Sarah" aunque tal vez si sea un arranque rápido en comparación con anteriores versiones de Windows.

Ahora ustedes pensaran "Y porque el titulo de la nota entonces?", pues permiteme contarte que este famoso arranque rapido es una farza!! porque cuando nosotros "apagamos" nuestra PC con windows 10 esta no realiza un apagado completo si no un tipo de hibernacion en una estado mas profundo con el fin de no cerrar windows por completo y asi al momento de volver a utilizar nuestra querida PC esta arrancara mas rapido :O.

Que nos dice esto?, bueno que simplemente es verdad el arranque mas rapido de windows 10 en relación a anteriores versiones pero lo hace a costa de una trampa al no mencionar que tu PC con windows no estara apagada por completo si no es un estado muy profundo de reposo.

Bueno y que pasa con esto? Tal vez tu puedas vivir con ello sabiendo este pequeño inconveniente, pero que pasa cuando utilizamos Linux en conjunto con Windows :S he aquí el gran problema pues al no cerrarse windows por completo al momento de arrancar las particiones que este utilizando nuestra PC no funcionaran en Linux incluyendo la misma partición en donde tenemos instalado Windows. Esto debido a que linux no es capaz de montar dichas particiones porque windows no las termino de usar correctamente o dicho de otra manera no las cierra adecuadamente ocasionando lo anterior dicho.

Pero que podemos hacer para que esto no suceda; Desactivando el arranque rápido de Windows 10. :D

Te dejo un link para que soluciones este problema si es que eres una victima mas de el famoso arranque rápido de windows.

https://goo.gl/qBciRi

Error a solucionar:

Error mounting /dev/sda2 at /media/debo/2290657790655279: Command-line `mount -t "ntfs" -o "uhelper=udisks2,nodev,nosuid,uid=1000,gid=1000,dmask=0077,fmask=0177" "/dev/sda2" "/media/debo/2290657790655279"' exited with non-zero exit status 14: Windows is hibernated, refused to mount. Failed to mount '/dev/sda2': Operation not permitted The NTFS partition is in an unsafe state. Please resume and shutdown Windows fully (no hibernation or fast restarting), or mount the volume read-only with the 'ro' mount option.



Tambien te paso el comando para apagar Windows al terminarlo de usar para que nuestro linux trabaje sin inconvenientes: shutdown /s /f /t 0 pulsas INTRO y el sistema se apaga por completo :D.

Estas listo para PHP 7




PHP7 tiene muchas mejoras en el lenguaje que hacen que tu código sea mucho más eficiente y limpio, sin embargo no todas las aplicaciones están listas para dar el gran paso en la versión por eso te recomendamos hacer una verificación primero antes de intentar cualquier migración.

PHPCompatibility es parte de los sniff para PHP_CodeSniffer que te permite verificar la compatibilidad de tu código PHP. La herramienta te permite validar si tu aplicación es compatible con versiones desde la versión 5 hasta la última liberada.

Puedes instalarla con composer simplemente agregando la librería en la sección "require-dev" y luego corriendo el comando update.

"require-dev": {
"squizlabs/php_codesniffer": "*",
"wimg/php-compatibility": "*",
"simplyadmire/composer-plugins" : "@dev",
"prefer-stable" : true
}

Puedes agregar opciones de configuración especifica para tu proyecto mediante un archivo xml.

<?xml version="1.0"?> 
<ruleset name="Custom ruleset">
<description> My rules for PHP_CodeSniffer</description> 
<!-- Run against the PHPCompatibility ruleset --> 
<rule ref="PHPCompatibility" /> 
<!-- Run against a second ruleset --> 
<rule ref="PSR2" /> 
</ruleset>

Puedes ver todas las opciones disponibles y el resto de la configuración en el sitio web.
Una herramienta muy útil para los desarrolladores de php que quieren actualizar sus proyectos y pasarlos a la nueva versión del lenguaje.
